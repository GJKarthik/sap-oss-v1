📋 Readiness Criteria Reference
🔧 DEV Criteria
Source Code — README + deps file (package.json / requirements.txt / pyproject.toml)
Server Impl — server.py / MCP server / engine running
Dockerfile — Containerizable build
Local Dev — docker-compose.yml or nginx.conf for local run
🧪 TEST Criteria
Unit/Integration Tests — *.spec.ts / *.test.ts / test_*.py files
SBOM — CycloneDX JSON in scripts/sbom-lineage/boms/
OpenAPI Spec — openapi.yaml in docs/api or service docs/api
🚀 PROD Criteria
Container — Dockerfile present (push-ready image)
K8s Manifest — deployment.yaml or statefulset.yaml in deploy/kyma/
APIRule — Kyma APIRule / Ingress for external access
NetPol / mTLS — NetworkPolicy or PeerAuthentication for zero-trust
Methodology: Readiness assessed by static analysis of the codebase at commit 176fb518 (main branch, April 2026). Scores derived from presence of: Dockerfiles, Kyma manifests (deploy/kyma/), test suites, CycloneDX SBOMs, OpenAPI specs, and ENDPOINTS documentation.

Key Gaps Identified: (1) TEST stage is the weakest across most services — only sac-webcomponents-ngx and odata-vocabularies have strong test coverage.
