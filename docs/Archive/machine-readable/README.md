# FinSight Archive Machine-Readable Exports

This directory contains AI-ready exports generated from:

- `docs/Archive/1_register.csv`
- `docs/Archive/2_stagingschema.csv`
- `docs/Archive/2_stagingschema_logs.csv`
- `docs/Archive/2_stagingschema_nonstagingschema.csv`
- `docs/Archive/3_validations.csv`

## Outputs

- `normalized/*.cleaned.csv` and `normalized/*.cleaned.jsonl`
  - Per-source cleaned rows with normalized column names and metadata.
- `finsight_records.jsonl` / `finsight_records.csv`
  - Unified records (one record per source row) with:
    - `record_id`, source metadata
    - normalized `fields`
    - synthesized row-level `text` for retrieval/LLM context
- `rag_chunks.jsonl` / `rag_chunks.csv`
  - Embedding-ready chunks (currently one chunk per record row).
- `rag_embedding_records.jsonl` / `rag_embedding_records.csv`
  - Chunk records with deterministic `embedding_id` and `text_sha256`.
- `rag_manifest.json`
- `rag_embedding_records_manifest.json`
- `schema_summary.json`
  - Header-row selection and original-to-normalized column mapping per source file.
- `quality_report.json`
  - Quality summary including mandatory coverage, normalization counts, and issue totals.
- `quality_issues.csv`
  - Record-level quality/normalization issues (e.g., placeholder replacement, datatype anomalies).
- `finsight_onboarding.odps.yaml`
  - ODPS 4.1 package descriptor for this data product.
- `odps_validation_report.json`
  - Validation report generated by schema + local link checks.
- `mangle/`
  - Mangle governance layer:
    - `facts.mg` (records, fields, mandatory definitions, quality issues)
    - `rules.mg` (derived governance checks)
    - `functions.mg` (helper predicates using functions)
    - `aggregations.mg` (aggregate quality metrics)
    - `manifest.json` (fact counts and inputs)
- `hana_phase0/`
  - Phase 0 baseline and HANA-target preparation package:
    - `phase0_contract_freeze.json` (frozen artifact hashes + baseline metrics)
    - `PHASE0_CONTRACT_FREEZE.md` (human-readable freeze summary)
    - `phase0_field_catalog.csv/json` (profile-driven field -> suggested HANA type map)
    - `phase0_hana_domains.yaml` (canonical domain/type definitions)
    - `phase0_target_schemas.yaml` (target schema blueprint)
    - `phase0_schema_skeleton.sql` (initial schema creation stub)
- `odata_phase2/`
  - Phase 2 OData vocabulary projection package:
    - `finsight_schema.edmx` (OData CSDL projection of target tables)
    - `finsight_column_annotations.json` (column-level Common/PersonalData term assignments)
    - `finsight_derived_checks.json` (data-cleaning-copilot check payload, when dependencies are present)
    - `phase2_validation_report.json` (coverage + term/check mapping report)
- `copilot_phase3/`
  - Phase 3 copilot integration package:
    - `phase3_validation_report.json` (copilot integration and table-load status)
    - `phase3_session_blueprint.json` (copilot/API command blueprint + starter prompts)
    - `PHASE3_COPILOT_BOOTSTRAP.md` (operator runbook)
