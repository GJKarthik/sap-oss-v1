\subsection{Executive Summary}

Trial Balance review is conducted during M+1 to M+7 across all Legal entities, 
Segment \& Group, to understand variance \& balance movements between periods. 
An AI model produces business insights and written commentaries based on GL transaction 
data analysis -- detecting anomalies, trends, unusual movements, new accounts, spikes/drops 
vs prior periods, and material balances.

\subsection{BSS Risk Assessment \& Integration Point}

The BSS Risk Assessment is conducted during M+12 to M+15 across all Legal entities, 
critical for CFO attestation and Risk/Audit committee reporting.

\begin{itemize}[noitemsep]
    \item \textbf{Risk Categories:} Substantiated at Risk, Unsubstantiated Unowned, Unsubstantiated Owned
    \item \textbf{AI Capabilities:} Highlight high-risk GL accounts, suggest risk ratings, detect anomalies in balances/ageing/adjustments
    \item \textbf{FTE Impact:} 10-12 FTEs current, 1 FTE savings
    \item \textbf{Scope:} In: BSS Risk Assessment. Out: FORTM, BSS Reconciliations
\end{itemize}

\textbf{Integration with TB Workflow:}

BSS Risk Assessment branches from the main TB workflow at the \textbf{UPDATE\_RISK} state. 
When risk scores from the TB analysis phase exceed the BSS materiality threshold, 
the workflow triggers a \textbf{BSS\_RA\_SUBPROCESS} that runs as a deferred sub-process 
(M+12 to M+15). Data flow: UPDATE\_RISK produces a risk register per LE; the BSS sub-process 
consumes this register plus historical ageing data to produce the CFO attestation pack. 
The integration is one-directional -- BSS results do not feed back into the current-month 
TB review but are available for the next cycle's anomaly detection baseline.

\section{Process Specification}

The TB review follows a formal BPMN workflow (source: Operation of Month End Close Controls -- 
Trial Balance Review -- GLO). The workflow comprises \textbf{27 states} (including 3 error-handling states), 
\textbf{5 exclusive gateways}, and spans \textbf{4 timeline phases}.

\begin{table}[h]
\centering
\caption{Timeline Phases}
\begin{tabular}{@{}L{4.5cm}C{2.5cm}L{7cm}@{}}
\toprule
\textbf{Phase} & \textbf{Timing} & \textbf{Description} \\
\midrule
Preparation & M-3 & Setup: roll forward file, update parameters and master data \\
Extraction & M-3 to M+5 & Extract TB from PSGL/S4 and load into variance analysis file \\
Extraction$\rightarrow$Analysis handshake & M-2 & UPDATE\_TB\_DETAILS completes; triggers CALCULATE\_VARIANCE \\
Analysis & M-2 to M+5 & Variance calculation, commentary collection, investigation, risk \\
Review & M+5 & Internal review, incorporate changes, send FORTM pack \\
\bottomrule
\end{tabular}
\end{table}

\textbf{Phase overlap:} Extraction (M-3 to M+5) and Analysis (M-2 to M+5) overlap because 
extraction runs per-LE on a rolling basis. Analysis begins as soon as the first batch of LEs 
complete extraction (M-2). The handshake state UPDATE\_TB\_DETAILS acts as the trigger: once TB 
details are loaded into the variance file for a given LE, that LE enters the analysis phase. 
LEs still awaiting extraction continue in parallel.

\subsection{State Machine}

\subsubsection{Preparation Phase (M-3)}

\begin{itemize}[noitemsep]
    \item \textbf{START} -- Prepare Schedule Work Day
    \item \textbf{ROLL\_FORWARD} -- Roll Forward Prior Month File [MS Excel, Shared Drive]
    \item \textbf{DOWNLOAD\_PSGL} -- Download PSGL Master Account File [PSGL, MS Excel]
    \item \textbf{UPDATE\_ACCOUNTS\_MASTER} -- Update Accounts Master Sheet [MS Excel]
    \item \textbf{UPDATE\_PARAMETERS} -- Update Parameters \& Comparatives [MS Excel]
\end{itemize}

\subsubsection{Extraction Phase (M-3 to M+5)}

\begin{itemize}[noitemsep]
    \item \textbf{EXTRACT\_TB} -- Extract Trial Balance [SC Bridge, MS Excel]
    \item \textbf{VALIDATE\_EXTRACTION} -- Verify row counts, checksums, and LE completeness [Automated]
    \item \textbf{UPDATE\_TB\_DETAILS} -- Update TB Details in Variance File [MS Excel, Shared Drive] EUC: EX/200/1579702/0001/000
\end{itemize}

\subsubsection{Error Handling (any phase)}

\begin{itemize}[noitemsep]
    \item \textbf{RETRY\_EXTRACTION} -- Retry failed PSGL/S4 API call with exponential backoff (max 3 retries, 30s/60s/120s) [Automated]
    \item \textbf{ESCALATE\_API\_FAILURE} -- Alert GFS Analyst after 3 failed retries; log to incident tracker; enable manual EUC override [MS Outlook, ITSM]
    \item \textbf{MANUAL\_OVERRIDE} -- GFS Analyst manually extracts data via EUC spreadsheet; flags LE for post-hoc reconciliation [MS Excel]
\end{itemize}

\subsubsection{Analysis Phase (M-2 to M+5)}

\begin{itemize}[noitemsep]
    \item \textbf{CALCULATE\_VARIANCE} -- Calculate Period-over-Period Variance [MS Excel]
    \item \textbf{FILTER\_VARIANCES} -- Filter by Materiality Criteria [MS Excel]
    \item \textbf{CHECK\_MATERIAL\_VARIANCES} [Gateway] -- Material? Yes$\rightarrow$SEEK\_COMMENTARIES | No$\rightarrow$CHECK\_UNEXPLAINED
    \item \textbf{SEEK\_COMMENTARIES} -- Request Explanations [MS Outlook]
    \item \textbf{RECEIVE\_COMMENTARIES} -- Collect Responses [Event]
    \item \textbf{UPDATE\_COMMENTARIES} -- Record in Variance File [MS Excel]
    \item \textbf{IS\_JOURNAL\_POSTED} [Gateway] -- Posted$\rightarrow$CHECK\_UNEXPLAINED | Required$\rightarrow$SEND\_REQUEST | Not Posted$\rightarrow$INVESTIGATE
    \item \textbf{SEND\_JOURNAL\_REQUEST} -- Request Posting [MS Outlook]
    \item \textbf{CHECK\_FURTHER\_INVESTIGATION} [Gateway] -- No$\rightarrow$CHECK\_UNEXPLAINED | Yes$\rightarrow$CHECK\_ENTRIES
    \item \textbf{CHECK\_ENTRIES\_STATUS} -- Follow Up with Teams [MS Outlook]
    \item \textbf{CHECK\_CONFIRMATION} [Gateway] -- By M+4? Yes$\rightarrow$CHECK\_UNEXPLAINED | No$\rightarrow$TRACK\_ENTRIES
    \item \textbf{TRACK\_ENTRIES} -- Track and Risk Analysis [MS Excel]
    \item \textbf{UPDATE\_RISK} -- Update Risk Assessment [MS Excel] (BSS RA sub-process trigger)
    \item \textbf{CHECK\_UNEXPLAINED} [Gateway] -- None$\rightarrow$PREPARE\_SUMMARY | Yes$\rightarrow$CATEGORIZE
    \item \textbf{CATEGORIZE\_UNEXPLAINED} -- Classify Unexplained Variances [MS Excel]
\end{itemize}

\subsubsection{Review Phase (M+5)}

\begin{itemize}[noitemsep]
    \item \textbf{PREPARE\_SUMMARY} -- Consolidate Analysis [MS Excel]
    \item \textbf{SEND\_INTERNAL\_REVIEW} -- Submit for Review [MS Outlook, Shared Drive]
    \item \textbf{REVIEW\_INCORPORATE} -- Incorporate Changes [MS Excel] EUC: EX/200/1579702/0002/000
    \item \textbf{SEND\_FORTM} -- Submit FORTM Pack [MS Outlook, Shared Drive]
    \item \textbf{END} -- TB Review Completed for Month
\end{itemize}

\subsection{Decision Flowchart}

The following diagram shows the 5 exclusive gateways and their branching logic. 
Error-handling states (dashed) can trigger from any extraction or API-dependent state.

\begin{lstlisting}
START -> ROLL_FORWARD -> DOWNLOAD_PSGL -> UPDATE_ACCOUNTS_MASTER
  -> UPDATE_PARAMETERS -> EXTRACT_TB -> VALIDATE_EXTRACTION
  -> UPDATE_TB_DETAILS -> CALCULATE_VARIANCE -> FILTER_VARIANCES

  [GW1: CHECK_MATERIAL_VARIANCES]
    |-- Material? YES --> SEEK_COMMENTARIES --> RECEIVE
    |                     --> UPDATE_COMMENTARIES
    |-- Material? NO  --------------------------> [GW5]

  [GW2: IS_JOURNAL_POSTED]
    |-- Posted     --------------------------> [GW5]
    |-- Required   --> SEND_JOURNAL_REQUEST --> [GW3]
    |-- Not Posted --> INVESTIGATE -----------> [GW3]

  [GW3: CHECK_FURTHER_INVESTIGATION]
    |-- No  ---------------------------------> [GW5]
    |-- Yes --> CHECK_ENTRIES_STATUS ---------> [GW4]

  [GW4: CHECK_CONFIRMATION]
    |-- By M+4? YES -------------------------> [GW5]
    |-- By M+4? NO  --> TRACK_ENTRIES
                        --> UPDATE_RISK ------> [GW5]

  [GW5: CHECK_UNEXPLAINED]
    |-- None -------> PREPARE_SUMMARY --> SEND_REVIEW
    |                   --> REVIEW_INCORPORATE --> SEND_FORTM --> END
    |-- Has unexpl --> CATEGORIZE_UNEXPLAINED --> PREPARE_SUMMARY

  --- Error path (from EXTRACT_TB / DOWNLOAD_PSGL) ---
  API_FAIL --> RETRY_EXTRACTION (x3)
           --> ESCALATE_API_FAILURE --> MANUAL_OVERRIDE

\textbf{DP-1 Threshold Feedback Loop:}

The ML-adaptive materiality threshold (DP-1) is trained on historical variance distributions 
per LE and Finance Caption. The feedback loop operates as follows: (1) At month-end, the current 
threshold classifies variances as material/non-material. (2) Analysts override classifications 
during the review (marking false positives/negatives). (3) Override data is stored as labelled 
training samples. (4) Monthly batch retrain updates the threshold model using the last 12 months 
of override history. (5) New thresholds are deployed for the next cycle with a comparison report 
showing threshold drift. Initial deployment uses the current country-specific static thresholds 
(BS: \$100M, PL: \$3M) as the baseline.
