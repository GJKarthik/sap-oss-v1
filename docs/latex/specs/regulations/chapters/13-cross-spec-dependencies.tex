\chapter{Cross-Specification Dependencies}
\label{ch:cross-spec}

This chapter documents how the Regulations and AI Governance specification
integrates with the other three SAP OSS specifications. All dependencies
are traceable to \texttt{docs/schema/registry.json} (v1.2.0).

% -----------------------------------------------------------------------------
\section{Architecture Position}
\label{sec:arch-position}

The Regulations specification occupies the \textbf{horizontal governance wrapper}
position in the four-domain architecture:

\begin{itemize}
  \item \textbf{Wraps}: Arabic AP (all AI-powered stages)
  \item \textbf{Wraps}: Trial Balance (all AI-powered stages)
  \item \textbf{Wraps}: Simula (training data governance)
  \item \textbf{Downstream}: Audit logs, compliance reports, conformance results
\end{itemize}

\section{MAS MGF Four Pillars Application}
\label{sec:pillar-application}

\subsection{Pillar 1: Assess \& Bound Risks}

Applied to all three operational domains:

\begin{itemize}
  \item \textbf{Arabic AP}: MCP tool allow-list (invoice processing tools only)
  \item \textbf{Trial Balance}: MCP tool allow-list (TB analysis tools only)
  \item \textbf{Simula}: Training data bounds (no PII, synthetic only)
\end{itemize}

\subsection{Pillar 2: Human Accountability}

Human approval checkpoints across domains:

\begin{itemize}
  \item \textbf{Arabic AP}: DP-AP-001 through DP-AP-004 (see Arabic Ch.13)
  \item \textbf{Trial Balance}: DP-001 through DP-005 (see TB Ch.13)
  \item \textbf{Simula}: Schema extraction approval, corpus sign-off
\end{itemize}

\subsection{Pillar 3: Technical Controls}

Cross-domain monitoring and testing:

\begin{itemize}
  \item Drift monitoring signals from all AI stages (Chapter 8)
  \item Emergent capability monitoring (Chapter 8)
  \item AI Verify/Moonshot conformance testing (Chapter 6)
\end{itemize}

\subsection{Pillar 4: End-User Responsibility}

Training and awareness programs:

\begin{itemize}
  \item AP clerk training for Arabic AP users (Chapter 9)
  \item Financial controller training for TB users (Chapter 9)
  \item Automation bias awareness across all domains
\end{itemize}

\section{Governance Wrapper Architecture}
\label{sec:wrapper-arch}

The Regulations domain acts as a horizontal wrapper around all AI-powered
stages in the other three specifications:

\begin{enumerate}
  \item All MCP tools are gated through the allow-list (Pillar 1)
  \item All material decisions require human approval (Pillar 2)
  \item All AI actions are monitored for drift/emergence (Pillar 3)
  \item All users receive training on automation bias (Pillar 4)
\end{enumerate}

This ensures consistent governance across the entire SAP OSS platform while
allowing each domain to focus on its specific functionality.
