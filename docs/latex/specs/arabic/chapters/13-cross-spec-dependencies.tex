\chapter{Cross-Specification Dependencies}
\label{ch:cross-spec}

This chapter documents how the Arabic AP Invoice Processing specification
integrates with the other three SAP OSS specifications. All dependencies
are traceable to \texttt{docs/schema/registry.json} (v1.2.0).

% -----------------------------------------------------------------------------
\section{Architecture Position}
\label{sec:arch-position}

The Arabic AP specification occupies the \textbf{document intake} position in
the four-domain architecture:

\begin{itemize}
  \item \textbf{Upstream}: None (entry point for invoice documents)
  \item \textbf{Downstream}: Trial Balance (posted AP records feed month-end close)
  \item \textbf{Horizontal}: Regulations (governance wrapper for all AI stages)
  \item \textbf{Training}: Simula (consumes Arabic schemas for training data generation)
\end{itemize}

\section{Regulations Integration}
\label{sec:reg-integration}

The Arabic AP pipeline is wrapped by the Regulations domain's MAS MGF framework:

\subsection{Pillar 1: Assess \& Bound Risks}

\begin{itemize}
  \item MCP tool allow-list defined in Chapter~\ref{ch:implementation}
  \item Reversibility: All AP postings support reversal
  \item Autonomy bounds: VAT verdicts require human approval for HOLD/REJECT
\end{itemize}

\subsection{Pillar 2: Human Accountability}

Human approval checkpoints (see Regulations Ch.3):
\begin{itemize}
  \item DP-AP-001: Invoice value above threshold
  \item DP-AP-002: New vendor first invoice
  \item DP-AP-003: VAT engine HOLD verdict
  \item DP-AP-004: OCR confidence below threshold
\end{itemize}

\subsection{Pillar 3: Technical Controls}

\begin{itemize}
  \item Drift monitoring: VAT engine accuracy tracked per Regulations Ch.8
  \item Emergent capability monitoring: OCR field extraction rates
  \item Conformance testing: AI Verify hooks per Regulations Ch.6
\end{itemize}

\subsection{Pillar 4: End-User Responsibility}

\begin{itemize}
  \item AP clerk training curriculum per Regulations Ch.9
  \item Automation bias awareness for VAT verdict review
\end{itemize}

\section{Simula Integration}
\label{sec:simula-integration}

The Simula Training Data Framework consumes Arabic AP schemas:

\begin{itemize}
  \item \textbf{invoice.schema.json} $\rightarrow$ Training examples for invoice field extraction
  \item \textbf{vat-checklist.schema.json} $\rightarrow$ Training examples for VAT classification
  \item \textbf{HANA metadata} $\rightarrow$ Schema extraction for query generation
\end{itemize}

See Simula specification Chapter 3 for schema extraction details.

% -----------------------------------------------------------------------------
\section{Trial Balance Integration}
\label{sec:tb-integration}

Arabic AP provides downstream data to Trial Balance:

\begin{itemize}
  \item Posted AP records serve as evidence for TB variance explanations
  \item AP journal entries appear in TB extract as liability movements
  \item Cross-reference: See TB specification Chapter 4 (Entity Parameters)
\end{itemize}
