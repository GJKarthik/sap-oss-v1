\chapter{Cross-Specification Dependencies}
\label{ch:cross-spec}

This chapter documents how the Simula Training Data Framework specification
integrates with the other three SAP OSS specifications. All dependencies
are traceable to \texttt{docs/schema/registry.json} (v1.2.0).

% -----------------------------------------------------------------------------
\section{Architecture Position}
\label{sec:arch-position}

The Simula specification occupies the \textbf{training infrastructure} position
in the four-domain architecture:

\begin{itemize}
  \item \textbf{Upstream}: Arabic AP (consumes invoice schemas)
  \item \textbf{Upstream}: Trial Balance (consumes TB/PL extract schemas)
  \item \textbf{Horizontal}: Regulations (training data governance)
  \item \textbf{Downstream}: Training JSONL $\rightarrow$ Model fine-tuning
\end{itemize}

\section{Regulations Integration}
\label{sec:reg-integration}

Simula training data generation is governed by Regulations:

\subsection{Pillar 1: Training Data Bounds}

\begin{itemize}
  \item No PII in generated training examples
  \item Synthetic data only---no production data extraction
  \item Complexity distribution balanced (easy/medium/hard)
\end{itemize}

\subsection{Pillar 2: Human Accountability}

\begin{itemize}
  \item Schema extraction requires DBA approval
  \item Training corpus sign-off by domain SME
  \item Double-critic verdicts logged for audit
\end{itemize}

\subsection{Pillar 3: Technical Controls}

\begin{itemize}
  \item Drift monitoring: Training data quality metrics per Regulations Ch.8
  \item SQL syntax validation gate (see Algorithm 3)
  \item Conformance testing: AI Verify hooks per Regulations Ch.6
\end{itemize}

\subsection{Pillar 4: Data Provenance}

\begin{itemize}
  \item Schema lineage tracking per Regulations Ch.7
  \item Training corpus versioning with SBOM integration
\end{itemize}

\section{Arabic AP Integration}
\label{sec:arabic-integration}

Simula consumes Arabic AP schemas for training data generation:

\begin{itemize}
  \item \textbf{invoice.schema.json} $\rightarrow$ Invoice field extraction training
  \item \textbf{vat-checklist.schema.json} $\rightarrow$ VAT category classification
  \item \textbf{Schema extraction} from HANA tables: BKPF, BSEG, LFA1
  \item Cross-reference: See Arabic AP specification Chapter 2 (Data Schema)
\end{itemize}

% -----------------------------------------------------------------------------
\section{Trial Balance Integration}
\label{sec:tb-integration}

Simula consumes TB schemas for training data generation:

\begin{itemize}
  \item \textbf{tb-extract.schema.json} $\rightarrow$ Account analysis training
  \item \textbf{pl-extract.schema.json} $\rightarrow$ P\&L commentary training
  \item \textbf{variance-record.schema.json} $\rightarrow$ Variance explanation training
  \item \textbf{Schema extraction} from HANA tables: ACDOCA, BSEG, SKB1
  \item Cross-reference: See TB specification Chapter 2 (Data Schema)
\end{itemize}
