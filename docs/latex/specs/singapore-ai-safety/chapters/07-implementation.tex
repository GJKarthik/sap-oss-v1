\section{Governance module architecture}

The governance module in \filepath{src/intelligence/ai-core-pal/governance/}
consists of three components:

\subsection{Identity module (\filepath{identity.py})}

Implements identity attribution per REG-MGF-2.1.2-002:

\begin{itemize}
  \item \texttt{AgentIdentity}: Dataclass for agent identity fields, validated
        against \filepath{agent-identity.schema.json}
  \item \texttt{RequestIdentity}: Dataclass for request identity fields, validated
        against \filepath{request-identity.schema.json}
  \item \texttt{IdentityEnvelope}: Combined envelope for governance tracking
  \item \texttt{IdentityProvider}: Factory for creating and validating identity
        envelopes
\end{itemize}

\subsection{Audit module (\filepath{audit.py})}

Implements durable audit trail per REG-MGF-2.3.3-002:

\begin{itemize}
  \item \texttt{AuditEvent}: Dataclass for audit events, validated against
        \filepath{audit-event.schema.json}
  \item \texttt{AuditReservation}: Reservation-before-action pattern
  \item \texttt{DurableAuditStore}: SQLite-backed persistent storage
  \item Support for reservation, completion, orphan detection
\end{itemize}

\subsection{Governed agent (\filepath{governed\_agent.py})}

Implements the full governance flow:

\begin{enumerate}
  \item Create identity envelope
  \item Validate envelope
  \item Check tool allow-list (deny-by-default)
  \item Check approval requirements
  \item Reserve audit BEFORE action
  \item Execute action
  \item Complete audit AFTER action
\end{enumerate}

\section{Integration points}

\subsection{MCP tool allow-list}

The governance engine maintains an explicit allow-list:

\begin{verbatim}
ALLOWED_TOOLS = frozenset({
    "pal_classification",
    "pal_regression",
    "pal_clustering",
    "pal_forecast",
    "pal_anomaly",
})
\end{verbatim}

Any tool not in this list is blocked (deny-by-default).

\subsection{Approval-required tools}

High-risk tools require human approval:

\begin{verbatim}
APPROVAL_REQUIRED_TOOLS = frozenset({
    "pal_train_model",
    "pal_delete_model",
    "hana_write",
    "model_deploy",
})
\end{verbatim}

\subsection{Audit database schema}

The SQLite audit database contains:

\begin{verbatim}
CREATE TABLE audit_events (
    event_id TEXT PRIMARY KEY,
    event_type TEXT NOT NULL,
    timestamp TEXT NOT NULL,
    request_id TEXT NOT NULL,
    correlation_id TEXT NOT NULL,
    agent_id TEXT NOT NULL,
    agent_type TEXT NOT NULL,
    status TEXT NOT NULL,
    duration_ms INTEGER,
    tokens_used INTEGER,
    error_code TEXT,
    action_json TEXT,
    approval_chain_json TEXT,
    request_identity_json TEXT NOT NULL,
    agent_identity_json TEXT NOT NULL,
    full_event_json TEXT NOT NULL,
    is_orphaned INTEGER DEFAULT 0
);
\end{verbatim}

\section{Testing and validation}

\subsection{Validation harness}

The validation harness at
\filepath{src/intelligence/ai-core-pal/tests/test\_clinerules\_contracts.py}
validates:

\begin{itemize}
  \item Structural completeness of rule packs
  \item Safety-critical flag inheritance (no weakening)
  \item Regulatory alignment requirements
  \item Cross-pack consistency
  \item Schema references
\end{itemize}

\subsection{Running validation}

\begin{verbatim}
# Run full validation
python src/intelligence/ai-core-pal/tests/test_clinerules_contracts.py

# Run with pytest
pytest src/intelligence/ai-core-pal/tests/test_clinerules_contracts.py -v
\end{verbatim}

\section{Deployment checklist}

Before deploying a governance-compliant agent:

\begin{enumerate}
  \item Verify all schemas exist and validate
  \item Verify rule packs pass validation harness
  \item Verify audit database is accessible
  \item Verify identity provider is configured
  \item Verify tool allow-list is correct for use case
  \item Calculate risk scores and verify production gate criteria
  \item Run minimum sample size tests per risk tier
  \item Confirm monitoring and alerting are active
\end{enumerate}
